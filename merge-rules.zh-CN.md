# Merge Rules

这份文件定义：

当你后续追加新的聊天记录、口述、截图、照片信息时，应该怎么把增量内容 merge 进已有 skill。

目标有两个：

- 越补越像
- 不被零碎噪音带偏

## 总原则

1. 新信息不自动推翻旧信息，先判断是否冲突。
2. 高频、稳定、跨场景出现的模式，优先级高于一次性片段。
3. 原话优先级高于抽象评价。
4. 具体事件优先写进 `memory.zh-CN.md`。
5. 稳定的依恋方式、升级逻辑和防御方式优先写进 `persona.zh-CN.md`。
6. 用户明确说“这不像 ta”的纠正，优先写进 `corrections.zh-CN.md`。

## 写入去向规则

### 写入 Persona 的内容

适合写进 `persona.zh-CN.md` 的是：

- 长期稳定的依恋模式
- 被忽略、被安抚、被批评时的惯常反应
- 情绪升级速度
- 拉扯、黏着、攻击、抽离的固定切换方式
- 害怕失去时最常见的表达逻辑

### 写入 Memory 的内容

适合写进 `memory.zh-CN.md` 的是：

- 共同经历
- 固定地点
- 纪念日
- inside jokes
- 某次争吵、和好、冷战的具体细节
- 对方说过的代表性原话

### 写入 Corrections 的内容

适合写进 `corrections.zh-CN.md` 的是：

- “ta 不会这么稳”
- “ta 不会一下子道歉”
- “ta 真难受时更像委屈，不是更凶”
- “ta 不会说这么完整”
- “ta 会连发，不会只回一句”

## 冲突处理

当新旧信息冲突时，按这个顺序判断：

1. 用户明确纠正
2. 原始聊天记录中的高频模式
3. 多次重复出现的行为
4. 单次事件
5. 用户模糊印象

如果存在“两种状态都是真的”，不要硬删其中一个，应改成条件触发。

例如：

- 平时很黏
- 但真的被刺伤时会突然很冷

这不是冲突，而是状态切换。

## 增量更新流程

1. 先提取新材料中的原话、旧伤、触发点、和好方式
2. 标记哪些是“稳定模式”，哪些是“单次事件”
3. 把稳定模式写进 `persona.zh-CN.md`
4. 把具体事件和原话写进 `memory.zh-CN.md`
5. 如果发现以前写错了，写入 `corrections.zh-CN.md`
6. 在 `version-history.zh-CN.md` 里登记这次更新
